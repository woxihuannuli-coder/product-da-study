
# 도닥톡 Product Analysis

## 도닥톡 소개
도닥톡은 복약 기록과 감정 기록을 기반으로 사용자의 자기관리 행동을 돕는 헬스케어 서비스입니다.  
이 프로젝트에서는 신규 사용자가 가입 후 7일 안에 다시 방문하고, 실제 복약 기록까지 이어지며, 그 행동을 반복하는지를 확인하는 데 초점을 맞췄습니다.

## 왜 이 지표를 사용했는가?
신규 사용자는 가입 직후 서비스의 모든 기능을 충분히 익히기 어렵다고 판단했습니다.  
따라서 초기 단계에서는 아래 3가지 흐름을 중심으로 초기 정착 여부를 확인하고자 했습니다.

1. 다시 방문하는가
2. 실제 복약 기록까지 이어지는가
3. 그 사용이 반복되는가

## 핵심 KPI
1. 가입 후 7일 재방문율
2. 가입 후 7일 첫 복약 기록 전환율
3. 가입 후 7일 복약 기록 유지율

## SQL_분석_파일
- `sql/dodaktok_revisit_7d.sql`
: 신규가입자의 초기 재방문 여부를 확인하기 위한 쿼리

- `sql/dodaktok_first_medication_conversion_7d.sql`
: 가입 후 7일 안에 실제 복약 기록까지 이어졌는지 확인하기 위한 쿼리

- `sql/dodaktok_medication_retention_7d.sql`
: 복약 기록이 꾸준히 이루어지고 있는지 확인하기 위한 쿼리
