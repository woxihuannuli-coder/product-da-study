# DodakTalk Product Analysis

## DodakTalk 소개

DodakTalk은 복약 기록과 감정 기록을 기반으로 사용자의 자기관리 행동을 돕는 헬스케어 서비스입니다.  
이 프로젝트에서는 신규 사용자가 가입 후 7일 안에 다시 방문하고, 실제 복약 기록까지 이어지며, 그 행동을 반복하는지를 확인하는 데 초점을 맞췄습니다.

## 분석 목표

이 프로젝트의 목표는 "복약 기록이 꾸준히 쌓이는 사용자"를 늘리기 위해, 가입 후 7일 동안 사용자의 초기 정착 여부를 진단하는 것입니다.  
특히, 복약을 시작했다가 금방 중단하는 사용자가 많은 특성을 고려해, 서비스 초반 행동 흐름을 수치로 파악하고자 했습니다.

## 왜 이 지표를 사용했는가?

신규 사용자는 가입 직후 서비스의 모든 기능을 충분히 익히기 어렵다고 판단했습니다.  
따라서 초기 단계에서는 아래 3가지 흐름을 중심으로 초기 정착 여부를 확인하고자 했습니다.

1. 다시 방문하는가
2. 실제 복약 기록까지 이어지는가
3. 그 사용이 반복되는가

## 핵심 KPI

1. 가입 후 7일 재방문율
2. 가입 후 7일 첫 복약 기록 전환율
3. 가입 후 7일 복약 기록 유지율

## 데이터 및 이벤트 가정

본 분석에서는 이벤트 로그에 `sign_up`, `app_open`, `medication_logged` 등의 이벤트가 존재한다고 가정했습니다.  
각 이벤트는 `user_id`와 `event_time`(UTC 기준 타임스탬프)을 포함하고 있으며,  
이를 기반으로 가입 시점부터 7일 동안의 재방문 여부, 첫 복약 기록 전환 여부, 복약 기록이 반복적으로 발생하는지 여부를 계산했습니다.

## SQL 분석 파일

- `sql/dodaktalk_revisit_7d.sql` : 신규 가입자의 초기 재방문 여부를 확인하기 위한 쿼리
- `sql/dodaktalk_first_medication_conversion_7d.sql` : 가입 후 7일 안에 실제 복약 기록까지 이어졌는지 확인하기 위한 쿼리
- `sql/dodaktalk_medication_retention_7d.sql` : 복약 기록이 꾸준히 이루어지고 있는지 확인하기 위한 쿼리

## 이 분석으로 확인하고자 한 것

세 가지 핵심 지표를 통해 다음과 같은 질문에 답하고자 했습니다.

- 온보딩과 알림 설계가 신규 사용자의 7일 내 재방문을 충분히 유도하고 있는가
- 가입 이후 실제 복약 기록으로 이어지는 전환 경로에 병목 지점은 어디에 있는가
- 복약 기록이 일회성 행동에 그치지 않고, 일정 기간 이상 반복되는 패턴을 보이는 사용자는 어느 정도 비중을 차지하는가

이 결과를 바탕으로, 초기 온보딩 플로우 개선이나 리마인드 메시지 전략 등 제품 레벨에서 어떤 가설을 세울 수 있을지 고민하는 것이 이 프로젝트의 최종 목표입니다.
