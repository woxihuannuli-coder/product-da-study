# analysis-sql-v1

## 공통 기준
- **분석대상(코호트)**: 2026년 4월 신규가입자 중, 분석 시점 기준 가입 후 7일 관찰이 완료된 사용자
- **코호트 진입 기준**: `signup_completed`
- **관찰 기간**: 각 사용자의 가입 시점 기준 7일
- **분석 목적**: 신규가입자의 초기 재방문, 핵심행동 진입, 반복행동 여부를 계산 가능한 형태로 정리한다.
- **제외 기준**: 테스트 계정, 가입 후 7일이 지나지 않은 사용자

## KPI 1. 가입 후 7일 재방문율

### 1) 정의
가입 후 7일 내 앱을 **서로 다른 날짜 기준 3일 이상** 재방문한 사용자 비율

### 2) 비즈니스 질문
신규가입자가 첫 일주일 안에 이 서비스를 다시 찾고 싶을 만큼 매력적으로 느끼는가?

### 3) 분모 계산 로직
2026년 4월 신규가입자 중, 분석 시점 기준 가입 후 7일 관찰이 완료된 사용자 수

### 4) 분자 계산 로직
분모 대상자 중 가입 후 7일 내 `app_opened`가 발생한 **서로 다른 날짜 수가 3일 이상**인 사용자 수

### 5) 필요한 이벤트/컬럼
- `user_id`
- `event_name`
- `event_time`
- `signup_completed`
- `app_opened`

### 6) 계산 로직 초안
1. `signup_completed` 이벤트를 기준으로 2026년 4월 신규가입 사용자와 가입 시점을 추출한다.
2. 이 중 분석 시점 기준 가입 후 7일 관찰이 완료된 사용자만 남긴다.
3. 각 사용자별로 가입 후 7일 내 발생한 `app_opened` 이벤트를 조회한다.
4. 같은 날짜의 중복 방문은 1일로 간주하고, 방문한 서로 다른 날짜 수를 계산한다.
5. 방문 날짜 수가 3일 이상인 사용자를 분자로 집계한다.
6. 분자 사용자 수를 전체 eligible 사용자 수로 나누어 가입 후 7일 재방문율을 계산한다.

### 7) 해석 포인트
- 재방문율이 높다면: 첫 인상과 초기 유입 경험이 비교적 긍정적일 가능성이 있다.
- 재방문율이 낮다면: 사용자에게 긍정적인 경험을 주지 못했음과 재방문을 유도하는 동기와 기능이 부족했을 가능성이 있다.

## KPI 2. 가입 후 7일 첫 복약 기록 전환율

### 1) 정의
가입 후 7일 내 복약 기록을 **1회 이상** 남긴 사용자 비율

### 2) 비즈니스 질문
신규가입자가 첫 일주일 안에 도닥톡의 핵심 행동인 복약 기록까지 실제로 진입하는가?

### 3) 분모 계산 로직
2026년 4월 신규가입자 중, 분석 시점 기준 가입 후 7일 관찰이 완료된 사용자 수

### 4) 분자 계산 로직
분모 대상자 중 가입 후 7일 내 `dose_checked`를 **1회 이상** 발생시킨 사용자 수

### 5) 필요한 이벤트/컬럼
- `user_id`
- `event_name`
- `event_time`
- `signup_completed`
- `dose_checked`

### 6) 계산 로직 초안
1. `signup_completed` 이벤트를 기준으로 2026년 4월 신규가입 사용자와 가입 시점을 추출한다.
2. 이 중 분석 시점 기준 가입 후 7일 관찰이 완료된 사용자만 남긴다.
3. 각 사용자별로 가입 후 7일 내 `dose_checked` 이벤트가 발생했는지 확인한다.
4. `dose_checked`가 1회 이상 발생한 사용자를 분자로 집계한다.
5. 분자 사용자 수를 전체 eligible 사용자 수로 나누어 가입 후 7일 첫 복약 기록 전환율을 계산한다.

### 7) 해석 포인트

- 전환율이 낮다면: 사용자가 첫 복약 기록까지 쉽게 도달하지 못하고 있을 가능성이 있다. 따라서 서비스 첫 진입 과정에서 어떤 지점이 사용자의 행동을 막고 있는지 점검할 필요가 있다
