# Event Log v1

## 1. signup_completed
- 정의: 사용자가 개인정보 입력과 가입 절차를 완료한 이벤트
- 왜 중요한가: 모든 사용자 행동의 시작점이며, 이후 Funnel 전환율과 초기 이탈을 분석하는 기준이 되기 때문이다.

## 2. character_selected
- 정의: 사용자가 온보딩 과정에서 캐릭터 선택을 완료한 이벤트
- 왜 중요한가: 초기 친밀감 형성 단계의 완료 여부를 확인하고, 온보딩 이탈 구간을 파악할 수 있기 때문이다.

## 3. tutorial_completed
- 정의: 사용자가 핵심 기능 안내를 확인하고 튜토리얼을 완료한 이벤트
- 왜 중요한가: 사용자가 기본 기능을 이해했는지 확인할 수 있으며, 초기 사용 장벽과 이탈 구간을 분석하는 데 필요하기 때문이다.

## 4. medication_registered
- 정의: 사용자가 복용 중인 약 정보를 OCR 또는 검색을 통해 등록한 이벤트
- 왜 중요한가: 복약 관리 기능의 시작점이며, 이후 복약 기록과 알림, 리포트 기능의 기준 데이터가 되기 때문이다.

## 5. dose_checked
- 정의: 사용자가 등록한 약에 대해 실제 복약 여부를 기록한 이벤트
- 왜 중요한가: 도닥톡의 핵심 행동이 실제로 발생했는지 확인할 수 있는 주요 지표이며, 복약 지속 여부 분석의 출발점이 되기 때문이다.

## 6. mood_logged
- 정의: 사용자가 아침, 점심, 저녁, 자기 전 중 특정 시간대의 기분 상태를 기록한 이벤트
- 왜 중요한가: 시간대별 기분 변화를 추적할 수 있으며, 복약 기록과 함께 사용자 상태 변화를 해석하는 핵심 데이터가 되기 때문이다.

## 7. report_viewed
- 정의: 사용자가 복약, 기분, 일기 기록을 바탕으로 생성된 리포트를 조회한 이벤트
- 왜 중요한가: 기록이 단순 입력에서 끝나지 않고, 자신의 상태를 해석하고 관리하는 행동으로 이어지는지 확인할 수 있기 때문이다.
