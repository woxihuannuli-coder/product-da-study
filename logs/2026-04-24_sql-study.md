# 4월 24일 시계열 SQL 공부

## 학습 목표

BigQuery 기준으로 `TIMESTAMP` 타입의 `event_time`을 다루는 기본 문법을 연습했다.

- UTC 기준으로 저장된 `event_time`을 KST 기준 `DATETIME`으로 변환하기
- KST 기준 날짜 부분만 추출하기
- KST 기준 시간(hour) 부분만 추출하기
- 날짜별/시간대별 이벤트 수 집계하기
- 이벤트명별, 사용자별 이벤트 수 집계하기
- 특정 이벤트만 필터링하기
- 고유 사용자 수 집계하기

---

## 가상 DB 테이블 구조

테이블명: `events`

| 컬럼명 | 설명 |
|---|---|
| `user_id` | 사용자 ID |
| `event_name` | 이벤트명 |
| `event_time` | 이벤트 발생 시각, TIMESTAMP 타입, UTC 기준 저장 |

`event_time`은 UTC 기준으로 저장되어 있다고 가정한다.
분석할 때는 한국 서비스 기준에 맞게 `Asia/Seoul` 기준으로 변환한다.

#문제 1. event_time을 KST 기준 DATETIME으로 변환하기

## 목표
UTC 기준으로 저장된 `event_time`을 KST 기준 `DATETIME`으로 변환한다.

## 내 답안
SELECT
user_id,
event_name,
event_time,
DATETIME(event_time,’Asia/Seoul’)as event_datetime_kst
FROM events;

## 정답

SELECT
  user_id,
  event_name,
  event_time,
  DATETIME(event_time, "Asia/Seoul") AS event_datetime_kst
FROM events;

##문제 2. event_time을 KST 기준 DATETIME으로 변환한 뒤, YYYY-MM-DD 날짜 부분만 추출하세요

##목표

 event_time을 KST 기준 DATETIME으로 변환한 뒤, YYYY-MM-DD 날짜 부분만 추출한다.

##내 답안

SELECT
user_id,
event_name,
DATE (DATETIME(event_time, ‘Asia/Seoul’))AS event_date_kst
FROM events;

## 정답

SELECT
  user_id,
  event_name,
  DATE(DATETIME(event_time, "Asia/Seoul")) AS event_date_kst
FROM events;

##문제 3. KST 기준 시간(hour) 추출하기

## 목표

시간대별 분석을 위해 0~23시 정보만 추출

##내 답안:

SELECT
user_id,
event_name,
EXTRACT(HOUR FROM DATETIME(event_time, 'Asia/Seoul')) AS event_hour_kst FROM events;

##최종 정답:

SELECT
  user_id,
  event_name,
  EXTRACT(HOUR FROM DATETIME(event_time, 'Asia/Seoul')) AS event_hour_kst
FROM events;

##문제 4. KST 기준 날짜별 이벤트 수 집계

##목표
날짜 단위로 묶어 총 이벤트 발생 건수 확인

##내 답안:

SELECT
event_date_kst,
COUNT(event_name),
DATE(event_time, 'Asia/Seoul') AS event_date_kst
FROM events GROUP BY event_date_kst;
오답 이유: event_date_kst라는 별칭을 정의함과 동시에 SELECT 첫 줄에서 호출하여 순서가 꼬임.

##최종 정답:

SELECT
  DATE(event_time, 'Asia/Seoul') AS event_date_kst,
  COUNT(event_name) AS event_cnt
FROM events
GROUP BY event_date_kst;

---

##문제 5. KST 기준 시간대별 이벤트 수 집계 (TRUNC 활용)

##목표
DATETIME_TRUNC를 활용해 시간 단위 집계

##내 답안:

SELECT event_hour_kst,
DATETIME_TRUNC(DATETIME(event_time, 'Asia/Seoul'), HOUR) AS event_hour_kst,
COUNT(event_name) AS event_cnt
FROM events
GROUP BY event_hour_kst;


##최종 정답:

SELECT
  DATETIME_TRUNC(DATETIME(event_time, 'Asia/Seoul'), HOUR) AS event_hour_kst,
  COUNT(event_name) AS event_cnt
FROM events
GROUP BY 1;

## 문제 6. KST 기준 이벤트명별 이벤트 수 집계

##내 답안:
SELECT event_name,
COUNT(*) AS event_cnt,

##문제 7. KST 기준 날짜별 고유 사용자 수(DAU) 집계

##내 답안:

SELECT DATETIME_TRUNC(DATETIME(event_time, 'Asia/Seoul'), DAY) AS event_date_kst,
COUNT(DISTINCT user_id)
FROM events
GROUP BY event_date_kst;

##최종 정답: (문법은 맞으나 가독성을 위해 DATE 함수 권장)

SELECT
  DATE(event_time, 'Asia/Seoul') AS event_date_kst,
  COUNT(DISTINCT user_id) AS dau
FROM events
GROUP BY 1;

##문제 9. KST 기준 날짜별 복약 체크('dose_checked') 수 집계

##목표
특정 이벤트에 대한 시계열 집계

##내 답안:

SELECT DATETIME(event_time, 'Asia/Seoul') AS event_date_kst,
count(*)
FROM events
WHERE event_name = 'dose_checked'
GROUP BY event_date_kst;


##최종 정답: (날짜별 집계이므로 DATE 함수 사용)

SELECT
  DATE(event_time, 'Asia/Seoul') AS event_date_kst,
  COUNT(*) AS dose_check_cnt
FROM events
WHERE event_name = 'dose_checked'
GROUP BY 1;

SELECT
