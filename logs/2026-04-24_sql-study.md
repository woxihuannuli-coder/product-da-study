# 4월 24일 시계열 SQL 공부

## 학습 목표

BigQuery 기준으로 `TIMESTAMP` 타입의 `event_time`을 다루는 기본 문법을 연습했다.

- UTC 기준으로 저장된 `event_time`을 KST 기준 `DATETIME`으로 변환하기
- KST 기준 날짜 부분만 추출하기
- KST 기준 시간(hour) 부분만 추출하기
- 날짜별/시간대별 이벤트 수 집계하기
- 이벤트명별, 사용자별 이벤트 수 집계하기
- 특정 이벤트만 필터링하기
- 고유 사용자 수 집계하기

---

## 가상 DB 테이블 구조

테이블명: `events`

| 컬럼명 | 설명 |
|---|---|
| `user_id` | 사용자 ID |
| `event_name` | 이벤트명 |
| `event_time` | 이벤트 발생 시각, TIMESTAMP 타입, UTC 기준 저장 |

`event_time`은 UTC 기준으로 저장되어 있다고 가정한다.  
분석할 때는 한국 서비스 기준에 맞게 `Asia/Seoul` 기준으로 변환한다.

---

## 문제 1. `event_time`을 KST 기준 `DATETIME`으로 변환하기

### 목표

UTC 기준으로 저장된 `event_time`을 KST 기준 `DATETIME`으로 변환한다.

### 내 답안

```sql
SELECT
  user_id,
  event_name,
  event_time,
  DATETIME(event_time, ’Asia/Seoul’) AS event_datetime_kst
FROM events;
```

### 최종 정답

```sql
SELECT
  user_id,
  event_name,
  event_time,
  DATETIME(event_time, "Asia/Seoul") AS event_datetime_kst
FROM events;
```

### 배운 점

- `DATETIME(event_time, "Asia/Seoul")`은 UTC 기준 `TIMESTAMP`를 KST 기준 `DATETIME`으로 변환한다.
- SQL에서는 `’Asia/Seoul’` 같은 스마트 따옴표보다 `"Asia/Seoul"`처럼 일반 따옴표를 사용하는 것이 안전하다.

## 문제 2. `event_time`을 KST 기준 `DATETIME`으로 변환한 뒤, `YYYY-MM-DD` 날짜 부분만 추출하기

### 목표

`event_time`을 KST 기준 `DATETIME`으로 변환한 뒤, `YYYY-MM-DD` 날짜 부분만 추출한다.

### 내 답안

```sql
SELECT
  user_id,
  event_name,
  DATE(DATETIME(event_time, ‘Asia/Seoul’)) AS event_date_kst
FROM events;
```

### 최종 정답

```sql
SELECT
  user_id,
  event_name,
  DATE(DATETIME(event_time, "Asia/Seoul")) AS event_date_kst
FROM events;
```

### 배운 점

- `DATE(DATETIME(event_time, "Asia/Seoul"))`은 UTC 시간을 KST 기준으로 변환한 뒤 날짜 부분만 추출한다.
- `DATE()`는 날짜와 시간 중에서 날짜 부분만 남길 때 사용한다.

## 문제 3. KST 기준 시간(hour) 추출하기

### 목표

시간대별 분석을 위해 KST 기준으로 0~23시 정보를 추출한다.

### 내 답안

```sql
SELECT
  user_id,
  event_name,
  EXTRACT(HOUR FROM DATETIME(event_time, 'Asia/Seoul')) AS event_hour_kst
FROM events;
```

### 최종 정답

```sql
SELECT
  user_id,
  event_name,
  EXTRACT(HOUR FROM DATETIME(event_time, "Asia/Seoul")) AS event_hour_kst
FROM events;
```

### 배운 점

- `EXTRACT()`는 날짜/시간 값에서 특정 부분만 뽑을 때 사용한다.
- 문법은 `EXTRACT(뽑을_단위 FROM 날짜시간값)` 순서다.

## 문제 4. KST 기준 날짜별 이벤트 수 집계하기

### 목표

KST 기준 날짜 단위로 묶어 총 이벤트 발생 건수를 확인한다.

### 내 답안

```sql
SELECT
  event_date_kst,
  COUNT(event_name),
  DATE(event_time, 'Asia/Seoul') AS event_date_kst
FROM events
GROUP BY event_date_kst;
```

### 오답 이유

- `event_date_kst`라는 별칭을 만들기 전에 `SELECT` 첫 줄에서 먼저 호출했다.
- `SELECT` 안에서 만든 별칭은 보통 `GROUP BY`에서는 사용할 수 있지만, 같은 `SELECT` 안에서 앞줄에 먼저 쓰면 안 된다.
- 집계값에는 `AS event_cnt`처럼 별칭을 붙이는 것이 좋다.

### 최종 정답

```sql
SELECT
  DATE(event_time, "Asia/Seoul") AS event_date_kst,
  COUNT(event_name) AS event_cnt
FROM events
GROUP BY event_date_kst;
```

### 배운 점

- 날짜별 집계는 `날짜 기준 컬럼 + COUNT()` 구조로 작성한다.
- BigQuery에서는 `TIMESTAMP`를 특정 시간대 기준 날짜로 바꿀 때 `DATE(event_time, "Asia/Seoul")`을 사용할 수 있다.

---

## 문제 5. KST 기준 시간대별 이벤트 수 집계하기

### 목표

`DATETIME_TRUNC`를 활용해 KST 기준 시간 단위로 이벤트 수를 집계한다.

### 내 답안

```sql
SELECT
  event_hour_kst,
  DATETIME_TRUNC(DATETIME(event_time, 'Asia/Seoul'), HOUR) AS event_hour_kst,
  COUNT(event_name) AS event_cnt
FROM events
GROUP BY event_hour_kst;
```

### 오답 이유

- `event_hour_kst`라는 별칭을 만들기 전에 `SELECT` 첫 줄에서 먼저 호출했다.
- 같은 컬럼 별칭을 두 번 쓰고 있다.

### 최종 정답

```sql
SELECT
  DATETIME_TRUNC(DATETIME(event_time, "Asia/Seoul"), HOUR) AS event_hour_kst,
  COUNT(event_name) AS event_cnt
FROM events
GROUP BY event_hour_kst;
```

### 배운 점

- `DATETIME_TRUNC(..., HOUR)`는 날짜/시간을 시간 단위로 잘라준다.
- 예를 들어 `2024-01-02 08:35:10`은 `2024-01-02 08:00:00`으로 정리된다.
- `GROUP BY 1`도 가능하지만, 초보 단계에서는 `GROUP BY event_hour_kst`처럼 별칭을 직접 쓰는 것이 더 읽기 쉽다.

## 문제 6. KST 기준 이벤트명별 이벤트 수 집계하기

### 목표

이벤트명별로 이벤트가 몇 건 발생했는지 집계한다.

### 내 답안

```sql
SELECT
  event_name,
  COUNT(*) AS event_cnt,
FROM events
GROUP BY event_name;
```

### 오답 이유

- `COUNT(*) AS event_cnt` 뒤에 쉼표가 있으면 안 된다.
- `SELECT`의 마지막 컬럼 뒤에는 쉼표를 붙이지 않는다.

### 최종 정답

```sql
SELECT
  event_name,
  COUNT(*) AS event_cnt
FROM events
GROUP BY event_name;
```

### 배운 점

- 이벤트명별 집계는 `GROUP BY event_name`을 사용한다.
- 이 문제는 시간 기준 분석이 아니므로 KST 변환이 필요하지 않다.

## 문제 7. KST 기준 날짜별 고유 사용자 수(DAU) 집계하기

### 목표

KST 기준 날짜별로 몇 명의 고유 사용자가 이벤트를 발생시켰는지 확인한다.

### 내 답안

```sql
SELECT
  DATETIME_TRUNC(DATETIME(event_time, 'Asia/Seoul'), DAY) AS event_date_kst,
  COUNT(DISTINCT user_id)
FROM events
GROUP BY event_date_kst;
```

### 최종 정답

```sql
SELECT
  DATE(event_time, "Asia/Seoul") AS event_date_kst,
  COUNT(DISTINCT user_id) AS dau
FROM events
GROUP BY event_date_kst;
```

### 배운 점

- `COUNT(DISTINCT user_id)`는 고유 사용자 수를 구할 때 사용한다.
- DAU는 날짜별 고유 사용자 수이므로 날짜 기준으로 `GROUP BY` 한다.
- 내 답안도 큰 방향은 맞지만, 날짜별 집계에서는 `DATE(event_time, "Asia/Seoul")`이 더 간결하다.

## 문제 8. KST 기준 특정 이벤트만 필터링하기

### 목표

`event_name`이 `dose_checked`인 이벤트만 조회하고, KST 기준 이벤트 시간을 함께 출력한다.

### 최종 정답

```sql
SELECT
  user_id,
  event_name,
  DATETIME(event_time, "Asia/Seoul") AS event_datetime_kst
FROM events
WHERE event_name = "dose_checked";
```

### 배운 점

- 특정 조건의 데이터만 조회할 때는 `WHERE`를 사용한다.
- 이 문제는 집계가 아니라 필터링 문제이므로 `GROUP BY`가 필요하지 않다.

---

## 문제 9. KST 기준 날짜별 복약 체크 수 집계하기

### 목표

`event_name`이 `dose_checked`인 이벤트만 대상으로, KST 기준 날짜별 복약 체크 수를 집계한다.

### 내 답안

```sql
SELECT
  DATETIME(event_time, 'Asia/Seoul') AS event_date_kst,
  COUNT(*)
FROM events
WHERE event_name = 'dose_checked'
GROUP BY event_date_kst;
```

### 오답 이유

- 목표는 “날짜별” 집계인데, `DATETIME()`을 사용하면 날짜와 시간이 모두 남는다.
- 날짜별로 묶으려면 `DATE()`를 사용해 날짜 부분만 남기는 것이 적절하다.
- `COUNT(*)`에도 `AS dose_check_cnt`처럼 별칭을 붙이는 것이 좋다.

### 최종 정답

```sql
SELECT
  DATE(event_time, "Asia/Seoul") AS event_date_kst,
  COUNT(*) AS dose_check_cnt
FROM events
WHERE event_name = "dose_checked"
GROUP BY event_date_kst;
```

### 배운 점

- 특정 이벤트만 대상으로 집계할 때는 `WHERE`로 먼저 필터링한다.
- 날짜별 집계에서는 날짜 기준 컬럼을 만들고 그 기준으로 `GROUP BY` 한다.

## 문제 10. KST 기준 사용자별 이벤트 수 집계하기

### 목표

각 사용자(`user_id`)가 전체 기간 동안 총 몇 번의 이벤트를 발생시켰는지 활동량을 확인한다.

### 내 답안

```sql
SELECT
  user_id,
  COUNT(event_name)
FROM events
GROUP BY user_id;
```

### 최종 정답

```sql
SELECT
  user_id,
  COUNT(event_name) AS event_cnt
FROM events
GROUP BY user_id;
```

### 배운 점

- 사용자별 집계는 `GROUP BY user_id`를 사용한다.
- 집계 결과에는 `AS event_cnt`처럼 별칭을 붙이는 것이 좋다.

## 오늘 자주 틀린 부분 정리

### 1. 스마트 따옴표 사용

잘못된 예:

```sql
DATETIME(event_time, ‘Asia/Seoul’)
```

올바른 예:

```sql
DATETIME(event_time, "Asia/Seoul")
```

---

### 2. SELECT 마지막 컬럼 뒤에 쉼표 붙이기

잘못된 예:

```sql
SELECT
  event_name,
  COUNT(*) AS event_cnt,
FROM events;
```

올바른 예:

```sql
SELECT
  event_name,
  COUNT(*) AS event_cnt
FROM events;
```

---

### 3. 만든 별칭을 같은 SELECT 안에서 먼저 사용하기

잘못된 예:

```sql
SELECT
  event_hour_kst,
  DATETIME_TRUNC(DATETIME(event_time, "Asia/Seoul"), HOUR) AS event_hour_kst,
  COUNT(*) AS event_cnt
FROM events
GROUP BY event_hour_kst;
```

올바른 예:

```sql
SELECT
  DATETIME_TRUNC(DATETIME(event_time, "Asia/Seoul"), HOUR) AS event_hour_kst,
  COUNT(*) AS event_cnt
FROM events
GROUP BY event_hour_kst;
```

---

### 4. 날짜별 집계인데 DATETIME 전체를 사용하기

날짜별 집계에서는 날짜만 남기는 것이 좋다.

```sql
DATE(event_time, "Asia/Seoul")
```

또는 아래처럼 쓸 수도 있다.

```sql
DATE(DATETIME(event_time, "Asia/Seoul"))
```

## 최종 정리

이번 연습에서 익힌 핵심 문법은 다음과 같다.

```sql
-- UTC TIMESTAMP를 KST 기준 DATETIME으로 변환
DATETIME(event_time, "Asia/Seoul")
```

```sql
-- UTC TIMESTAMP를 KST 기준 날짜로 변환
DATE(event_time, "Asia/Seoul")
```

```sql
-- UTC TIMESTAMP를 KST 기준 DATETIME으로 변환한 뒤 날짜 부분만 추출
DATE(DATETIME(event_time, "Asia/Seoul"))
```

```sql
-- KST 기준 시간(hour) 추출
EXTRACT(HOUR FROM DATETIME(event_time, "Asia/Seoul"))
```

```sql
-- KST 기준 시간 단위로 자르기
DATETIME_TRUNC(DATETIME(event_time, "Asia/Seoul"), HOUR)
```

```sql
-- 전체 이벤트 수 집계
COUNT(*)
```

```sql
-- 고유 사용자 수 집계
COUNT(DISTINCT user_id)
```
