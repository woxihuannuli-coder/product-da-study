# 포켓몬 데이터 분석 SQL 문제 풀이

## 주요 학습 내용

- `JOIN`: 여러 테이블의 정보를 연결
- `WHERE`: 조건에 맞는 행만 필터링
- `IN`: 여러 값 중 하나에 해당하는지 확인
- `COUNT`: 행 수 집계
- `COUNT(DISTINCT)`: 중복 제거 후 집계
- `GROUP BY`: 기준별 집계
- `ORDER BY`: 정렬

---

## 문제 1. 트레이너가 보유한 포켓몬 종류별 수 계산

### 목표

`status`가 `Active`, `Training`인 경우를 기준으로, 트레이너들이 현재 보유한 포켓몬 종류별 수를 계산한다.

### 내 답안

```sql
SELECT
kor_name
count(tp.id) as pokemon_cnt
  FROM(
  SELECT
  id,
  trainer_id,
  pokemon_id,
  status
  FROM `basic.trainer_pokemon`

  WHERE (status = 'Active') or (status = 'Training')
) AS tp
LEFT JOIN `basic.pokemon` AS p
ON tp.pokemon_id = p.id
GROUP BY p.kor_name
ORDER BY pokemon_cnt DESC
```

## 최종 정답
```sql
SELECT
  p.kor_name,
  COUNT(tp.id) AS pokemon_cnt
FROM `basic.trainer_pokemon` AS tp
LEFT JOIN `basic.pokemon` AS p
  ON tp.pokemon_id = p.id
WHERE tp.status IN ('Active', 'Training')
GROUP BY p.kor_name
ORDER BY pokemon_cnt DESC;
```

### 왜 틀렸는지
- 맞았지만, `IN ('Active', 'Training')`으로 쓰는 게 더 깔끔하다.


## 문제 2. 각 트레이너가 가진 Grass 타입 포켓몬 수 계산
## 목표

각 트레이너가 보유한 포켓몬 중 type1 = 'Grass'인 포켓몬 수를 계산한다.

## 내 답안
```sql
SELECT 
tp.*,
p.type1
FROM 
(SELECT
id,
pokemon_id,
trainer_id
FROM `basic.trainer_pokemon`) AS tp
LEFT JOIN `basic.pokemon` AS p
ON tp.pokemon_id = p.id
WHERE type1 = 'Grass'
GROUP BY tp.trainer_id
```

## 최종 정답
```sql
SELECT
  tp.trainer_id,
  COUNT(tp.pokemon_id) AS grass_pokemon_cnt
FROM `basic.trainer_pokemon` AS tp
LEFT JOIN `basic.pokemon` AS p
  ON tp.pokemon_id = p.id
WHERE p.type1 = 'Grass'
GROUP BY tp.trainer_id
ORDER BY tp.trainer_id;
```

### 왜 틀렸는지
- 문제는 “트레이너별 몇 마리인지”를 구하는 건데, 처음 답안은 `tp.*`까지 가져와서 너무 많은 정보를 같이 보여줬다.
- 이 문제는 `trainer_id`와 `COUNT(...)`만 있으면 된다.

## 문제 3. 자신의 고향에서 포켓몬을 포획한 트레이너 수 계산
## 목표

트레이너의 hometown과 포켓몬 포획 위치 location이 같은 경우를 찾아, 해당 트레이너 수를 계산한다.

## 내 답안
```sql
SELECT DISTINCT
 COUNT(DISTINCT tp.trainer_id) AS trainer_cnt

FROM `basic.trainer_pokemon` AS TP
LEFT JOIN `basic.trainer` AS T
  ON TP.trainer_id = T.id
WHERE T.hometown = TP.location
```

## 최종 정답
```sql
SELECT
  COUNT(DISTINCT tp.trainer_id) AS trainer_cnt
FROM `basic.trainer_pokemon` AS tp
LEFT JOIN `basic.trainer` AS t
  ON tp.trainer_id = t.id
WHERE t.hometown = tp.location;
```


## 문제 4. Master 등급 트레이너들이 가장 많이 보유한 포켓몬 타입 계산
## 목표

achievement_level = 'Master'인 트레이너들이 현재 보유한 포켓몬 중 어떤 type1을 가장 많이 보유하고 있는지 계산한다.

## 내 답안
```sql
SELECT 
P.type1,
  COUNT(R.pokemon_id) AS pokemon_count
FROM
(SELECT
*
FROM `basic.trainer` AS T
LEFT JOIN `basic.trainer_pokemon` AS TP
ON t.id = TP.trainer_id
WHERE achievement_level = 'Master' AND STATUS IN ('Active','Training'))AS R
LEFT JOIN `basic.pokemon` AS P
ON R.pokemon_id = p.id
GROUP BY P.TYPE1 
```

## 최종 정답
```sql
SELECT
  p.type1,
  COUNT(tp.pokemon_id) AS pokemon_count
FROM `basic.trainer` AS t
LEFT JOIN `basic.trainer_pokemon` AS tp
  ON t.id = tp.trainer_id
LEFT JOIN `basic.pokemon` AS p
  ON tp.pokemon_id = p.id
WHERE t.achievement_level = 'Master'
  AND tp.status IN ('Active', 'Training')
GROUP BY p.type1
ORDER BY pokemon_count DESC;
```

### 왜 틀렸는지
- 제일 많은 수를 기준으로 보려면 많은 순으로 정렬해야 해서 `ORDER BY pokemon_count DESC`가 필요하다.

## 문제 5. INCHEON 출신 트레이너들이 보유한 1세대, 2세대 포켓몬 수 계산
## 목표

hometown = 'Incheon'인 트레이너들이 보유한 포켓몬 중 generation이 1 또는 2인 포켓몬 수를 트레이너별, 세대별로 계산한다.

## 내 답안
```sql
SELECT
R.trainer_id,
generation,
count(*) as cnt
FROM
(SELECT
TP.*,
T.hometown
FROM `basic.trainer_pokemon`AS TP
LEFT JOIN `basic.trainer` AS T
ON TP.trainer_id = T.id)AS R
LEFT JOIN `basic.pokemon` AS P                         
ON R.pokemon_id = P.id
WHERE (hometown = 'Incheon')AND(p.generation IN(1,2))
GROUP BY R.trainer_id, p.generation
ORDER BY R.trainer_id
```

## 최종 정답
```sql
SELECT
  tp.trainer_id,
  p.generation,
  COUNT(*) AS cnt
FROM `basic.trainer_pokemon` AS tp
LEFT JOIN `basic.trainer` AS t
  ON tp.trainer_id = t.id
LEFT JOIN `basic.pokemon` AS p
  ON tp.pokemon_id = p.id
WHERE t.hometown = 'Incheon'
  AND p.generation IN (1, 2)
GROUP BY tp.trainer_id, p.generation
ORDER BY tp.trainer_id;
```
