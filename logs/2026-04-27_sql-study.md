# 포켓몬 데이터 분석 SQL 문제 풀이

## 주요 학습 내용

- `CASE WHEN`: 다중 조건에 따른 값 분류
- `IF`: 단일 조건에 따른 이진 분류
- `IN`: 여러 값 중 하나에 해당하는지 확인
- `BETWEEN`: 범위 조건 작성
- 날짜 비교: 기준일 이후/이전 데이터 분류
- Subquery & `GROUP BY`: 분류된 값을 기준으로 집계

## 문제 1. Rock & Ground 타입 통합 분류

### 목표

`type1` 또는 `type2`가 `Rock`, `Ground`인 포켓몬을 `Rock&Ground`로 통합 분류하는 새로운 컬럼을 만든다.

### 내 답안

```sql
SELECT
  *,
  CASE
    WHEN (type1 IN ('Rock', 'Ground')) OR (type2 IN ('Rock', 'Ground')) THEN "Rock&Ground"
    ELSE type1
  END AS new_type1
FROM `basic.pokemon`
WHERE type2 IN ('Rock', 'Ground');
```

### 최종 정답

```sql
SELECT
  *,
  CASE
    WHEN type1 IN ('Rock', 'Ground') OR type2 IN ('Rock', 'Ground') THEN 'Rock&Ground'
    ELSE type1
  END AS new_type1
FROM `basic.pokemon`;
```

### 배운 점

- `IN ('Rock', 'Ground')`는 `Rock` 또는 `Ground`에 해당하는지 확인할 때 사용한다.
- `WHERE`는 행을 필터링하는 역할이다.
- 전체 포켓몬을 대상으로 새 분류 컬럼을 만들고 싶다면 `WHERE`를 빼는 것이 더 적절하다.

## 문제 2. 스피드 기준 속도 카테고리 생성

### 목표

포켓몬의 `speed`가 70 이상이면 `빠름`, 그렇지 않으면 `느림`으로 표시하는 새로운 컬럼 `speed_category`를 만든다.

### 내 답안

```sql
SELECT
  *,
  IF(speed >= 70, "빠름", "느림") AS speed_category
FROM `basic.pokemon`;
```

### 최종 정답

```sql
SELECT
  *,
  IF(speed >= 70, '빠름', '느림') AS speed_category
FROM `basic.pokemon`;
```

### 배운 점

- 조건이 하나일 때는 `IF()`를 사용하면 된다.
- `IF(조건, 참일 때 값, 거짓일 때 값)` 순서로 작성한다.
- 문자열은 작은따옴표 또는 큰따옴표를 사용할 수 있지만, SQL에서는 작은따옴표를 일관되게 쓰는 것이 좋다.

## 문제 3. 주요 타입 한국어 번역 및 기타 분류

### 목표

`type1` 값이 `Water`, `Fire`, `Electric`이면 각각 `물`, `불`, `전기`로 변환하고, 그 외 타입은 `기타`로 분류하는 새로운 컬럼 `type_korean`을 만든다.

### 내 답안

```sql
SELECT
  id,
  kor_name,
  type1,
  CASE
    WHEN type1 = 'Water' THEN '물'
    WHEN type1 = 'Fire' THEN '불'
    WHEN type1 = 'Electric' THEN '전기'
    ELSE "기타"
  END AS type_korean
FROM `basic.pokemon`;
```

### 최종 정답

```sql
SELECT
  id,
  kor_name,
  type1,
  CASE
    WHEN type1 = 'Water' THEN '물'
    WHEN type1 = 'Fire' THEN '불'
    WHEN type1 = 'Electric' THEN '전기'
    ELSE '기타'
  END AS type_korean
FROM `basic.pokemon`;
```

### 배운 점

- 조건이 여러 개일 때는 `CASE WHEN`을 사용한다.
- `ELSE`는 위 조건에 해당하지 않는 나머지 값을 처리한다.

## 문제 4. 총점(`total`) 기반 등급 분류

### 목표

포켓몬의 `total` 값을 기준으로 `Low`, `Medium`, `High` 등급으로 분류하는 새로운 컬럼 `total_val`을 만든다.

분류 기준은 다음과 같다.

| 조건 | 분류 |
|---|---|
| `total <= 300` | Low |
| `301 <= total <= 500` | Medium |
| `total >= 501` | High |

### 내 답안

```sql
SELECT
  id,
  kor_name,
  CASE
    WHEN total >= 501 THEN 'High'
    WHEN total BETWEEN 301 AND 500 THEN 'Medium'
    ELSE 'Low'
  END AS total_val
FROM `basic.pokemon`;
```

### 최종 정답

```sql
SELECT
  id,
  kor_name,
  total,
  CASE
    WHEN total >= 501 THEN 'High'
    WHEN total BETWEEN 301 AND 500 THEN 'Medium'
    ELSE 'Low'
  END AS total_val
FROM `basic.pokemon`;
```

### 배운 점

- 범위 조건에서는 경계값이 빠지지 않게 주의해야 한다.
- 큰 값부터 조건을 쓰면 분류 기준이 더 명확해질 수 있다.
- 결과를 검증하기 위해 기준이 되는 원본 컬럼 `total`도 함께 조회하는 것이 좋다.

## 문제 5. 배지 개수별 트레이너 수준 집계

### 목표

트레이너의 `badge_count`를 기준으로 `Beginner`, `Intermediate`, `Advanced`로 분류한 뒤, 각 분류별 트레이너 수를 집계한다.

분류 기준은 다음과 같다.

| 조건 | 분류 |
|---|---|
| `badge_count <= 5` | Beginner |
| `6 <= badge_count <= 8` | Intermediate |
| `badge_count >= 9` | Advanced |

### 내 답안

```sql
SELECT
  badge_val,
  COUNT(badge_val) AS trainer_count
FROM (
  SELECT
    id,
    name,
    badge_count,
    CASE
      WHEN badge_count >= 9 THEN 'Advanced'
      WHEN badge_count BETWEEN 6 AND 8 THEN 'Intermediate'
      ELSE 'Beginner'
    END AS badge_val
  FROM `basic.trainer`
)
GROUP BY badge_val
ORDER BY badge_val DESC;
```

### 최종 정답

```sql
SELECT
  badge_val,
  COUNT(*) AS trainer_count
FROM (
  SELECT
    id,
    name,
    badge_count,
    CASE
      WHEN badge_count >= 9 THEN 'Advanced'
      WHEN badge_count BETWEEN 6 AND 8 THEN 'Intermediate'
      ELSE 'Beginner'
    END AS badge_val
  FROM `basic.trainer`
)
GROUP BY badge_val
ORDER BY trainer_count DESC;
```

### 배운 점

- 전체 행 수를 세는 목적이면 `COUNT(*)`가 더 일반적이다.
- `ORDER BY badge_val DESC`는 분류명 기준 정렬이고, `ORDER BY trainer_count DESC`는 인원수 기준 정렬이다.

## 문제 6. 포획 날짜 기준 신규/기존 데이터 분류

### 목표

트레이너가 포켓몬을 포획한 날짜 `catch_date`가 `2023-01-01` 이후이면 `Recent`, 그렇지 않으면 `Old`로 분류하는 새로운 컬럼 `catch_level`을 만든다.

### 내 답안

```sql
SELECT
  trainer_id,
  catch_date,
  IF(catch_date >= '2023-01-01', 'Recent', 'Old') AS catch_level
FROM `basic.trainer_pokemon`;
```

### 최종 정답

```sql
SELECT
  trainer_id,
  catch_date,
  IF(catch_date >= '2023-01-01', 'Recent', 'Old') AS catch_level
FROM `basic.trainer_pokemon`;
```

### 배운 점

- 날짜 컬럼은 `'YYYY-MM-DD'` 형식의 날짜 문자열과 비교할 수 있다.
- 기준일이 명확할 때는 `EXTRACT(YEAR FROM catch_date)`보다 `catch_date >= '2023-01-01'`처럼 날짜 전체를 비교하는 것이 더 정확하다.
